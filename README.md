# pravo-ritorika · «Скилл риторики для юриста»

Скилл-агент для проверки и усиления **убедительности** правовой позиции на материале практики РФ. Парный к [`pravo-logika`](https://github.com/sky-magenta/pravo-logika): логика проверяет, следует ли вывод, риторика — примет ли его конкретный адресат. Аппарат — классическая судебная риторика: статусная теория (выбор линии спора), диспозиция (порядок аргументов), учение о трёх средствах убеждения, техника опровержения. Все референсы на русском.

> Проект [Damascus Ink](https://damascus-ink.ru); автор — основатель платформы [Moire AI](https://moire-ai.tech).

## Что делает

- **Стресс-тест** — сильнейшая добросовестная контрпозиция (steelman) и карта уязвимостей подачи. Только диагностика, текст не переписывается.
- **Статус** — выбор линии позиции по статусной теории: спорить о факте, квалификации, оценке или процедуре, что доступно на каждой линии и чем за неё платить.
- **Структура** — разбор диспозиции меморандума или выступления: части, порядок аргументов, работа начала и конца; перестановка с обоснованием.
- **Адаптация** — переупаковка одной позиции под конкретного адресата: суд, нотариус, госорган, оппонент в переговорах, клиент.

## Границы

Скилл усиливает подачу, а не содержание. Три правила во всех режимах:

1. **Нормы и практику по памяти не подставляет.** Где сильный ход зависит от конкретной нормы или позиции суда — флаг `[требует проверки: норма / практика]`, проверка за человеком.
2. **Только честная аргументация.** Искажение фактов, подмена тезиса, давление на адресата вне предмета спора — вне скилла. Усиливается подача обоснованной позиции: структура, порядок, акценты, адресация.
3. **Валидность не проверяет.** Логические дефекты помечаются флагом `[→ pravo-logika]` и отправляются в парный скилл. Усиливать невалидный вывод запрещено.

## Устройство

```
pravo-ritorika/
├─ SKILL.md                    # режимы, форматы отчётов, маршрутизация, границы
├─ references/
│  ├─ status.md                # статусная теория: выбор линии позиции
│  ├─ dispositio.md            # структура текста и порядок аргументов
│  ├─ audience.md              # модели адресатов
│  ├─ appeals.md               # логос / этос / пафос
│  ├─ refutation.md            # steelman, стресс-тест, работа с возражениями
│  └─ rhetoric-errors.md       # каталог риторических дефектов подачи
├─ commands/
│  ├─ stress-test.md           # /pravo-ritorika:stress-test
│  ├─ status.md                # /pravo-ritorika:status
│  ├─ structure.md             # /pravo-ritorika:structure
│  └─ adapt.md                 # /pravo-ritorika:adapt
└─ README.md
```

## Установка

Скопируйте репозиторий в корень скиллов (имя папки должно совпадать с полем `name` в `SKILL.md`):

| Инструмент | Путь |
| --- | --- |
| Claude Code | `~/.claude/skills/pravo-ritorika/` |
| Cursor | `~/.cursor/skills/pravo-ritorika/` |
| OpenAI Codex | `~/.codex/skills/pravo-ritorika/` |

## Вызов

- Слэш-команды: `/pravo-ritorika:stress-test`, `/pravo-ritorika:status`, `/pravo-ritorika:structure`, `/pravo-ritorika:adapt`.
- Обычный запрос: «усиль позицию», «разбей мою позицию», «на чём строить защиту», «проверь структуру аргументации», «переупакуй для суда».
- Автоматически — агент подгружает скилл, если запрос совпадает с `description` в `SKILL.md`.

Связка с `pravo-logika`: сначала валидность, потом убедительность. Рабочий цикл над позицией — `/pravo-logika:review` → правка → `/pravo-ritorika:status` → `/pravo-ritorika:structure` → `/pravo-ritorika:stress-test`.
