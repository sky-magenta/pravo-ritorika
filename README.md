# pravo-ritorika · «Скилл риторики для юриста»

[![ci](https://github.com/sky-magenta/pravo-ritorika/actions/workflows/ci.yml/badge.svg)](https://github.com/sky-magenta/pravo-ritorika/actions/workflows/ci.yml)

Скилл-агент для проверки и усиления **убедительности** письменной правовой позиции на материале практики РФ: меморандумы, процессуальные документы, претензии, письма, заключения. Парный к [`pravo-logika`](https://github.com/sky-magenta/pravo-logika): логика проверяет, следует ли вывод, риторика — примет ли его конкретный адресат. Аппарат — классическая судебная риторика: статусная теория (выбор линии спора), диспозиция (порядок аргументов), учение о трёх средствах убеждения, техника опровержения. Все референсы на русском.

> Проект [Damascus Ink](https://damascus-ink.ru); автор — основатель платформы [Moire AI](https://moire-ai.tech). Об авторе — в разделе [«Об авторе и проекте»](#об-авторе-и-проекте).

## Что делает

- **Стресс-тест** — сильнейшая добросовестная контрпозиция (steelman) и карта уязвимостей подачи. Только диагностика, текст не переписывается.
- **Статус** — выбор линии позиции по статусной теории: спорить о факте, квалификации, оценке или процедуре, что доступно на каждой линии и чем за неё платить.
- **Структура** — разбор диспозиции меморандума, процессуального документа или письма: части, порядок аргументов, работа начала и конца, навигация письменного текста; перестановка с обоснованием.
- **Адаптация** — переупаковка одной позиции под конкретного адресата: суд, нотариус, госорган, оппонент в переговорах, клиент.

Как это выглядит на живом тексте — сквозной стресс-тест в [`docs/example.md`](docs/example.md); каталог дефектов подачи — [`docs/cheatsheet.md`](docs/cheatsheet.md); готовые запросы — [`docs/prompts.md`](docs/prompts.md).

## Границы

Скилл усиливает подачу, а не содержание. Три правила во всех режимах:

1. **Нормы и практику по памяти не подставляет.** Где сильный ход зависит от конкретной нормы или позиции суда — флаг `[требует проверки: норма / практика]`, проверка за человеком.
2. **Только честная аргументация.** Искажение фактов, подмена тезиса, давление на адресата вне предмета спора — вне скилла. Усиливается подача обоснованной позиции: структура, порядок, акценты, адресация.
3. **Валидность не проверяет.** Логические дефекты помечаются флагом `[→ pravo-logika]` и отправляются в парный скилл. Усиливать невалидный вывод запрещено.

## Установка

**Как папка-скилл** — скопируйте репозиторий в корень скиллов (имя папки должно совпадать с полем `name` в `SKILL.md`):

| Инструмент    | Путь                               |
| ------------- | ---------------------------------- |
| Claude Code   | `~/.claude/skills/pravo-ritorika/` |
| Cursor        | `~/.cursor/skills/pravo-ritorika/` |
| OpenAI Codex  | `~/.codex/skills/pravo-ritorika/`  |

**Как плагин (Claude Code)** — репозиторий является маркетплейсом Claude Code:

```
/plugin marketplace add sky-magenta/pravo-ritorika
/plugin install pravo-ritorika@damascus-ink
```

Проверка — `/plugin`. Плагин подключает и скилл (`SKILL.md` + `references/`), и слэш-команды `/pravo-ritorika:stress-test`, `/pravo-ritorika:status`, `/pravo-ritorika:structure`, `/pravo-ritorika:adapt`.

## Вызов

- Слэш-команды: `/pravo-ritorika:stress-test`, `/pravo-ritorika:status`, `/pravo-ritorika:structure`, `/pravo-ritorika:adapt` (готовые шаблоны — в [`docs/prompts.md`](docs/prompts.md)).
- Обычный запрос: «усиль позицию», «разбей мою позицию», «на чём строить защиту», «проверь структуру аргументации», «переупакуй для суда».
- Автоматически — агент подгружает скилл, если запрос совпадает с `description` в `SKILL.md`.

## Экосистема Damascus Ink

Семейство открытых скиллов для юриста (право РФ, один автор, MIT) — каждый закрывает свой слой работы над письменной позицией:

| Скилл | Слой | Статус |
| --- | --- | --- |
| [`pravo-logika`](https://github.com/sky-magenta/pravo-logika) | **Валидность**: следует ли вывод — субсумпция, толкование, однозначность договора, разбор нормы, сравнение позиций | опубликован |
| **`pravo-ritorika`** | **Убедительность**: примет ли адресат — линия спора, порядок аргументов, адаптация, стресс-тест | этот репозиторий |
| [`pravo-grammatika`](https://github.com/sky-magenta/pravo-grammatika) | **Форма**: как написано — синтаксическая неоднозначность, пунктуация, область действия, модальность | опубликован |

Связка логики и риторики: сначала валидность, потом убедительность. Рабочий цикл над позицией — `/pravo-logika:review` → правка → `/pravo-ritorika:status` → `/pravo-ritorika:structure` → `/pravo-ritorika:stress-test`. Риторика валидность не проверяет (флаг `[→ pravo-logika]`), логика не оценивает подачу — слои не пересекаются.

## Статус

Версия 0.2.0. Предмет — письменная юридическая речь (устная подача вне охвата); в диспозиции — жанры письменных документов и инструменты письменного текста (резюме первой страницы, рубрикация, таблицы, «стена текста» в каталоге дефектов). Четыре режима (стресс-тест, статус, структура, адаптация), у каждого свой формат отчёта в `SKILL.md`. Шесть референсов: статусная теория, диспозиция, модели адресатов, триада средств убеждения, техника опровержения (включая каталог недобросовестных приёмов оппонента — распознать и честно обезвредить), каталог дефектов подачи. Мультиинструментные манифесты (`.claude-plugin` / `.codex-plugin` / `.cursor-plugin`) и маркетплейс Claude Code; CI прогоняет проверку манифестов и валидатор набора E2E-тестов на каждый push. Есть набор E2E-тестов (`evals/`, 121 условный пример: 85 defect / 36 clean) на все четыре режима и разделы каталога дефектов, включая M&A / корпоративные сценарии, многодефектные тексты и корректные примеры на отсутствие ложных срабатываний; набор целиком прогнан на модели (см. `evals/README.md`). Лендинг — [`docs/index.html`](docs/index.html) (в стиле Damascus Ink, самодостаточная страница). Каталог дефектов — [`docs/cheatsheet.md`](docs/cheatsheet.md), готовые запросы — [`docs/prompts.md`](docs/prompts.md).

Лендинг проекта: [sky-magenta.github.io/pravo-ritorika](https://sky-magenta.github.io/pravo-ritorika/).
